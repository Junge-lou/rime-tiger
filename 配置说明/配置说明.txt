虎码 Rime 配置说明
==================

这份说明给第一次接触本配置的人看。想调候选数、样式、按键，先看第四节。
根目录里的 *.schema.yaml、*.custom.yaml 不要移动，Rime 会按这些文件名读取配置。
同目录有两份示例文件：
- 《示例.schema.yaml》：一份带中文注释的 schema 示例。
- 《示例.custom.yaml》：一份带中文注释的 custom 示例。
这两份只放在配置说明里给人看，Rime 不会读取。


一、相比原版整理了什么
--------------------------

1. 常用字和全字集改回同一方案内切换
   tiger / tigress 底层加载全字集词典，默认用 core2022 过滤到常用字。
   按 Ctrl+H 或 Ctrl+Shift+H 可在“常用字”和“全字集”之间切换。
   extended_char 关为常用字，开为全字集；不写 reset，避免新窗口重置。
   字集状态已加入 save_options，并由 option_sync 做跨窗口同步。

2. 全字集不再作为独立方案
   默认方案列表只显示 tiger / tigress / PY_c。
   tiger / tigress 使用各自的 .extended 词典入口，常用字和全字集由同一个开关切换。
   旧用户如果之前停在已删除旧方案，部署后重新选择一次 tiger 或 tigress 即可。

3. 虎码方案共享公共底座
   大部分按键、滤镜、符号、反查、空码处理都放在 tiger_base.schema.yaml。
   虎码共用的用户补丁放在 tiger_base.custom.yaml。
   修改公共配置时，不需要在多个文件里重复改。

4. 词库版保留加词、删词、调序
   tigress 启用用户加词 Lua。
   tiger 是单字方案，不写入词库用户词。

5. 方案名保持简洁
   切换方案时只显示方案名；字数保留在各方案 description 里。


二、方案和字集怎么选
------------------

1. tiger
   虎码单字
   底层字典：全字集 99144 字
   默认常用字过滤，适合只打单字。

2. tigress
   虎码词库
   底层字典：全字集 99144 字
   默认常用字过滤
   适合日常打词，支持加词、删词、调序。

3. PY_c
   拼音方案

推荐：
- 日常输入用 tigress。
- 只想打单字用 tiger。
- 需要生僻字、扩展字时，在 tiger 或 tigress 里按 Ctrl+H 切到全字集。


三、配置文件分工
----------------

每个真实方案都有两类文件：

1. *.schema.yaml
   方案本体。这里放方案名、词典入口、处理流程。

2. *.custom.yaml
   用户补丁。这里放候选数、样式、按键、开关。

一般优先改 *.custom.yaml。只有要改方案名、词典入口、处理流程时，再改
*.schema.yaml。

Rime 的方案字段叫 schema。tiger_base.schema.yaml 里也有 schema_id、name、
description，方便阅读和继承；不要把 tiger_base 加入方案列表。


四、继承关系
------------

schema 关系：

  tiger_base.schema.yaml
  ├─ tiger.schema.yaml               单字用户方案
  └─ tigress.schema.yaml             词库用户方案

custom 关系：

  tiger_base.custom.yaml
  ├─ tiger.custom.yaml               单字用户方案补丁
  └─ tigress.custom.yaml             词库用户方案补丁

读法：
- 上面的文件先加载。
- 下面的文件继承上面的配置，并覆盖自己的差异。


五、想改配置，改哪个文件
------------------------

1. 虎码方案都生效
   改 tiger_base.custom.yaml。

   适合：候选数量、候选编号、外观样式、通用按键。

2. 只改单字方案：tiger
   改 tiger.custom.yaml。

3. 只改词库方案：tigress
   改 tigress.custom.yaml。

4. 只改方案名或词典入口
   改对应的 *.schema.yaml。

常见例子：
- 想把候选数从 9 改成 5：改 tiger_base.custom.yaml。
- 只想词库版启用某个样式：改 tigress.custom.yaml。
- 只想词库改一个特殊按键：改 tigress.custom.yaml。
- 想改方案在切换菜单里的名字：改对应的 *.schema.yaml。


六、常用字和全字集怎么来的
--------------------------

单字版：
- tiger 使用 tiger.extended。
- 默认用 core2022 过滤为常用字；切到全字集时不过滤。

词库版：
- tigress 使用 tigress.extended。
- 默认用 core2022 过滤为常用字；切到全字集时不过滤。

全字集包含常用字。当前统计：
- 常用字：9767 字
- 全字集：99144 字


七、加词、删词、调序
--------------------

启用范围：
- tigress

未启用：
- tiger

快捷键：
- Ctrl+; 进入加词
- Ctrl+' 进入删词/屏蔽候选
- Ctrl+方向键 调整候选顺序
- macOS 可用 Ctrl+Option+方向键

用户词和调序记录写入 tigress.user.dict.yaml。


八、不要动这些
--------------

1. 不要把 *.schema.yaml / *.custom.yaml 移进文件夹。
   Rime 在根目录按文件名读取它们。

2. 不要把 tiger_base 加进 default.custom.yaml 的 schema_list。
   tiger_base 是公共底座，放进方案列表会让切换菜单出现一个不该选的配置项。

3. 不要删除 tiger_base.schema.yaml / tiger_base.custom.yaml。
   虎码方案都依赖它们。

4. 不要把加词 Lua 加到 tiger。
   单字方案保持单字逻辑，用户词管理交给词库方案。


九、改完怎么生效
----------------

改完配置后重新部署 Rime。

macOS 鼠须管：
从输入法菜单选择“重新部署”。

Windows 小狼毫：
右键输入法图标，选择“重新部署”。

部署失败时，先检查刚改过的 YAML 缩进，尤其是冒号后面的空格和列表缩进。
